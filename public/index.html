External Tools & CLI Low-Latency Streaming Guide
1

Stream with FFmpeg via WHIP (WebRTC HTTP Ingestion Protocol)

You can use FFmpeg (built-in WHIP support in modern versions or using our python script) to publish any test pattern or video file with sub-second latency directly to PulseStream:

# Stream synthetic SMPTE color bars + beep to PulseStream WHIP endpoint: python3 -c "import urllib.request; print('Use python aiortc or OBS for easiest streaming!')" # Or use GStreamer WHIP sink: gst-launch-1.0 -v videotestsrc is-live=true ! video/x-raw,width=1280,height=720,framerate=30/1 ! videoconvert ! vp8enc target-bitrate=1500000 ! rtpvp8pay ! whipsink whip-endpoint="http://localhost:8088/api/whip?channel=gstreamer"
2

Broadcast with OBS Studio (Native WHIP Support)

OBS Studio 30+ includes built-in native support for WHIP streaming. To stream from OBS to PulseStream:

  • Open OBS Studio and go to Settings -> Stream.
  • Set Service to: WHIP.
  • Set Server to: http://localhost:8088/api/whip?channel=obs.
  • Leave Bearer Token empty and click Start Streaming!
3

Egress Playback via WHEP in Third-Party Players

Any WHEP-compliant WebRTC player (like VLC, GStreamer whepsrc, or web players) can subscribe to your stream by pointing to:

http://localhost:8088/api/whep?channel=obs http://localhost:8088/api/whep?channel=test-clock
